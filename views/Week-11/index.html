COMP3100 portfolio entry #8

Where we are now

So, we are in the middle of week 11. I was able to complete the iOS registration, and the app is now open for beta participation, in both marketplaces. However, we haven't had many users try it. That is because we've kept a bit silent, and people are unaware they can use their app. However, the interest is there.

Other than this, we are also preparing for the TechLauncher showcase. We have a lot of resources available already, from our participation in ANU open day. Overall, we are not stressed about this. The only thing to do is to create and submit the poster we'll have to use. I'm currently working on it, and I'll talk about this on the contribution part of this entry.

Finally, the last thing we need to do as a team is to prepare for our final project review. This is three weeks away from now, so we have decided to focus on exams and other responsibilities first, and then start working on that.

My contributions this week

Again, I worked a lot on documentation this week. For starters, I created a document containing all the tasks that need to be completed for the project to be maintainable. The document can be seen to the right. Our goal is to allocate these tasks to each team member and complete them before the start of week 13, so around the end of October. I believe I've covered everything required so that people that are not familiar with the project can understand our platform, should they need to work on it.

In addition, I went through our existing documentation and brought it up to date. As an example, we have a glossary document that contains all the important source code files, and their role. This wasn't touched since July, so it was a bit outdated. In addition, our notes from tutorial meetings were all over the place, so I collected them and moved them all to one folder, and sorted them alphabetically. This process will not only be helpful for future members of the PairFit team, but it will also help us prepare for the final project review.

And last but not least, I also started working on a draft of the poster we need to prepare for next week. I've posted our current draft down below, but it is not yet complete. In any case, this process was much easier, because of all the marketing resources we have collected. In addition, Jarrod has been extremely helpful, and has provided some poster mock-ups he did in the past. I've been combining these, and I've been adding to them, in order to ensure that the final poster we submit complies with the poster requirements, as in the TechLauncher website. Not only will the poster be useful for attracting the attention of others in the showcase, but we can use it in the future to start advertising the startup around the ANU.

Closing thoughts

Overall, not too much work this week. I've had time to commit to my other responsibilities, which is always nice.

It does look like the Pairfit chapter of TechLauncher is getting close to it's end. What remains now is to start marketing the product, but Jarrod and AJ seem to have this responsibility. As such, my role in the team is coming to a close.

Next week's entry will be my last one. It's format will also be different. It will be more like a reflection, instead of a work portfolio, given that there aren't any other contributions I can talk about.
