COMP3100 portfolio entry #7

Copy of our (now obsolete) project plan

Where we are now

Week 10, and that means two more weeks left for TechLauncher, and therefore, this chapter of the project. At the moment, everyone's tasks are become increasingly monotonous, and mostly revolve around wrapping up the project. Currently, we are in the process of releasing the beta for iOS and using the feedback we got from the Android release last week to tweak the app. Officially, we are never touching functionality again, unless it's something small, like adding a prompt when logging out of the app, and so on.

What is left to do is to start documenting the project, for any teams that might potentially work on it. This was suggested by Richard Jones, the project's tutor, and it is indeed something that we've put on hold for a while. During last semester, we were much more "formal" in bookkeeping, commenting, and managing the "admin" stuff a bit better. I won't lie, I think we got a bit lazy in semester two. Hence, we have to rush to complete documentation now.

My contributions this week

This week's tasks were just fixing a couple of issues that Android users had detected. They were far too specific to be worth mentioning, and they were fixed within minutes. There wasn't much programming involved this week, at least for me.

However, this doesn't mean that I didn't contribute to the project. This week's goal was to push and release the iOS beta, and I was responsible for doing so, while the rest of the team was working on other things.

So, for starters, I had to compile the application in Xcode and handle the necessary signatures and certificates. A while back, I had to work on Apple push notifications, so I had already generated the necessary files required to verify our app, before releasing it in the iOS app store. In addition, the knowledge from COMP3310 (Computer Networks), which I did last semester, helped immensely. I was able to produce the required binaries in a short amount of time.

The real issue came with uploading the app to the store, however. I'm not really experienced in this field, so I tried going through Apple's guidelines, to figure out what needs to be done. As it turns out, Apple's documentation is a bit of a mess. The layout is not exactly straightforward, and the structure of the guide made finding answers to the questions I had extremely tedious. I ended up deciding to write this entry on my portfolio first, and giving up on uploading the app. My plan is to resume this task tomorrow.

Some other tasks I want to do is to start updating the documentation. As I've mentioned above, it's something that needs to be done, for when the project changes hands. Also, the final presentation for TechLauncher is in about a month, so it would be wise to start collecting artifacts as early as possible, given that exams are also coming up. Hence, I feel like shifting my focus to the management of the project is a reasonable thing to do.

Closing thoughts

Well, we did it. The app is out there and it's stable. Hopefully, tomorrow will be the iOS release date, and then we can start wrapping things up.

Things should start to slow down now, with the app being complete in terms of functionality. I'll add one more entry to my portfolio next week, and then I'll do a reflection on what I've learned and what skills I've acquired from the course. Given that this is a work portfolio, I think that reflecting on what I got out from this project would be not only helpful for any potential employers reading this message, but for me as well.

Until next time!
